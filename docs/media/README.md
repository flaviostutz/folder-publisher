## Weekly Operating Rhythm

Use this routine every campaign week from 08/04/2026 onward:

1. Monday: review previous metrics, choose one narrow use case, and prepare screenshots.
2. Tuesday: adapt one core asset for the week's primary platform.
3. Wednesday: publish the main post at 14:00 UTC and stay available for responses.
4. Thursday: distribute the same idea to one community or secondary platform.
5. Friday: summarize questions, objections, clicks, and GitHub star changes.

## Platform Frequency

- X: 1 launch thread in week 1, then 1 follow-up post every 1 to 2 weeks
- LinkedIn: 1 founder post in week 1, then 1 post every 2 weeks
- Dev.to: 1 launch article, then 1 deeper technical article every 3 to 4 weeks
- Reddit: 1 post in launch week, then reuse only when there is a new proof point or example
- Hacker News: 1 Show HN in launch week; wait for substantial product or traction updates before another submission
- Discord / Slack communities: 2 posts in launch week, then 1 targeted post per week in the most responsive communities

## Success Checks By Phase

- End of 10/04/2026: confirm which message produced the most clicks to GitHub
- End of 16/04/2026: confirm which platform produced the best stars-per-visit ratio
- End of 30/04/2026: decide which use case becomes the primary narrative for May
- End of 27/05/2026: review total stars gained, top-performing channel, best-performing CTA, and which new media assets must be created for the next cycle
